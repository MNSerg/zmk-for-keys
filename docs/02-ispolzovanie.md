# Использование прошивки и клавиатуры

Как пользоваться Corne v3 на этой прошивке: подключение к ПК, OLED, слои, правая половина, чем править раскладку.

Связанный документ: [Настройка файлов прошивки](01-nastrojka.md).

---

## 1. Что прошивать

| Файл из Actions | Плата |
|-----------------|-------|
| `corne_v3_left*.uf2` | Левый nice!nano |
| `corne_v3_right*.uf2` | Правый nice!nano |
| `settings_reset*.uf2` | Сброс настроек (временная прошивка) |

Процедура:

1. Отключите батарею (по возможности).
2. Дважды быстро нажмите RESET → появляется диск `NICENANO`.
3. Скопируйте `.uf2` → плата перезагрузится сама.
4. Для сброса: сначала `settings_reset` на половине, дождитесь перезагрузки/диска, затем снова обычный left/right UF2.

---

## 2. Подключение к компьютеру

### USB (проще для отладки)

1. Воткните **левую** половину в USB.
2. Клавиатура должна печатать сразу (имя устройства — **Corne**).
3. Правую в USB к ПК **для набора не подключают** — она не HID-клавиатура, только peripheral.

### Bluetooth

1. Питание левой (батарея или USB).
2. На ПК: найти **Corne** → сопряжение.
3. До 5 профилей BLE. Переключение — слой **ADJ** (зажмите нижний `mo 3` / крайний большой палец слева от середины правой половины — см. keymap):

| Клавиша (слой ADJ) | Действие |
|--------------------|----------|
| `&bt BT_SEL 0…3` | Профиль 0–3 |
| `&bt BT_CLR` | Очистить текущий профиль |
| `&out OUT_USB` / `OUT_BLE` / `OUT_TOG` | Принудительно USB / BLE / переключить |

Если ПК «не видит» клавиатуру после перепрошивки — `settings_reset` на левой, затем снова left UF2 и новое сопряжение.

## 3. Связь левой и правой половины

Половины связываются **между собой по BLE** (не через ПК).

### Как понять, что соединены

На OLED левой половины сейчас **нет** флага `R:OK` (кастомный экран отключён — ломал загрузку).

Проверка связи:

- Нажатия на **правой** появляются на ПК → split работает.
- Правая **не** должна отображаться отдельным Bluetooth-устройством на ПК.

### Правая плата сейчас вне PCB

Это нормально для проверки:

1. Правый nice!nano должен быть под питанием (USB или батарея).
2. Прошивка `corne_v3_right` должна быть залита.
3. Обе половины — из **одной** сборки Actions.
4. Нажмите клавишу на правой (если есть матрица) или смотрите, печатает ли ПК с правой стороны.

Если половины не видят друг друга:

1. `settings_reset` → снова left + right UF2.
2. Держите половины рядом (BLE).
3. Убедитесь, что прошили именно **right** на правый MCU (не left дважды).

## 4. Что показывает OLED

Сейчас используется **встроенный** экран ZMK (без `R:OK`):

| Элемент | Смысл |
|---------|--------|
| USB / BLE + номер профиля | Выход на ПК |
| Батарея | Заряд левой |
| Слой внизу | `DEF` / `LOW` / `RAI` / `ADJ` (или номер) |

Слои (имена в keymap):

| № | Имя | Как попасть |
|---|-----|-------------|
| 0 | DEF | Базовый |
| 1 | LOW | Удерживать Space (`&lt 1`) |
| 2 | RAI | Удерживать Enter (`&lt 2`) — трекбол = скролл |
| 3 | ADJ | Удерживать `&mo 3` (большой палец) |

---

## 5. Слои и полезные клавиши

Кратко по смыслу (точные позиции — в `config/corne_v3.keymap`):

- **DEF** — буквы, home-row mods (A/S/D/F и J/K/L/;).
- **LOW** — цифры, RGB toggle, bootloader, ext_power, кнопки мыши.
- **RAI** — стрелки/nav, numpad-ish, скролл трекболом.
- **ADJ** — Bluetooth-профили и USB/BLE output.

Home-row mods: короткое нажатие = буква, удержание ≈ 200 ms = модификатор. Если «медленно печатаются» именно A/S/D/F — это hold-tap, не USB. Параметр `tapping-term-ms` у `hm` в keymap.

Обычный антидребезг сейчас **5 ms** (раньше было 20 ms).

## 6. Чем настраивать слои на ПК

ZMK Studio в этой сборке **выключен** (`CONFIG_ZMK_STUDIO=n`), поэтому «живого» редактора по USB пока нет.

### Рекомендуемый путь (как устроена эта прошивка)

1. Открыть репозиторий на GitHub / локально.
2. Править `config/corne_v3.keymap` (и дубликат в щите, если пользуетесь им).
3. Push → дождаться Actions → прошить оба UF2 (или только left, если меняли только keymap/central).

Визуальный редактор keymap-файла (без Studio):

- [Keymap Editor (nickcoutsos)](https://nickcoutsos.github.io/keymap-editor/) — загружаете репозиторий/файл, правите слои графически, сохраняете `.keymap` обратно в git.

Документация по синтаксису:

- [ZMK Keymap](https://zmk.dev/docs/keymaps)
- [Behaviors](https://zmk.dev/docs/keymaps/behaviors)
- [Bluetooth](https://zmk.dev/docs/features/bluetooth)

### Если захотите ZMK Studio позже

1. В `corne_v3_left.conf` / общем conf: `CONFIG_ZMK_STUDIO=y` (и при необходимости studio-транспорт).
2. Пересобрать.
3. Подключить левую по USB → [ZMK Studio](https://zmk.studio/) в браузере.

Studio удобен для слоёв «на лету», но усложняет отладку boot — поэтому сейчас выключен.

### Что НЕ нужно

- QMK Toolbox / VIA / Vial — это другой стек (QMK), к этой прошивке не подходит.
- Отдельная утилита «только для ZMK слоёв» кроме Studio / keymap-editor / правки файла — по сути не требуется.

Полезные дополнения на ПК (не обязательны):

- [ZMK Battery Center](https://github.com/kot149/zmk-battery-center) — заряд split в трее (нужны флаги battery proxy/fetching).

## 7. RGB

- Вкл/выкл: слой LOW → `&rgb_ug RGB_TOG`.
- Цепочка: 6 underglow, затем 21 per-key на половине.
- Если светят только первые LED — смотрите пайку/цепь, не софт счётчика (уже 27).

---

## 8. Трекбол (правая в PCB)

- Движение курсора на слое DEF (и других без scroll-mapper).
- На слое **RAI** (hold Enter) — скролл.
- Кнопка трекбола — ЛКМ (через split input).
- Настройка CPI/scale — см. [01-nastrojka.md](01-nastrojka.md).

---

## 9. Чеклист «всё живое»

1. Левая по USB печатает.
2. OLED: батарея + слой (`DEF` и т.д.), не белый шум.
3. Правая под питанием → клавиша справа печатает на ПК.
4. BLE: отключить USB левой (на батарее) → ПК видит Corne.
5. (Опционально) трекбол двигает курсор; RAI — скролл.

### Если «белый шум» / ПК не видит клавиатуру

Кастомный OLED-экран **отключён**. Нужна сборка **после** коммита отката на built-in.

1. Actions → последний зелёный Build → `firmware.zip`.
2. `settings_reset*.uf2` на левую.
3. `corne_v3_left*.uf2`.
4. Переткнуть USB.

Если снова шум — прошейте `corne_v3_left_bare*.uf2` (без OLED). Если bare работает, а полный left нет — пишите (тогда отключим OLED в основной прошивке).
