# Использование прошивки и клавиатуры

Как пользоваться Corne v3 на этой прошивке: подключение к ПК, OLED, слои, правая половина, чем править раскладку.

Связанный документ: [Настройка файлов прошивки](01-nastrojka.md).

---

## 1. Что прошивать

| Файл из Actions | Плата |
|-----------------|-------|
| `corne_v3_left*.uf2` | Левый nice!nano |
| `corne_v3_right*.uf2` | Правый nice!nano |
| `settings_reset*.uf2` | Сброс настроек (временная прошивка) |

Процедура:

1. Отключите батарею (по возможности).
2. Дважды быстро нажмите RESET → появляется диск `NICENANO`.
3. Скопируйте `.uf2` → плата перезагрузится сама.
4. Для сброса: сначала `settings_reset` на половине, дождитесь перезагрузки/диска, затем снова обычный left/right UF2.

---

## 2. Подключение к компьютеру

### USB (проще для отладки)

1. Воткните **левую** половину в USB.
2. Клавиатура должна печатать сразу (имя устройства — **Corne**).
3. Правую в USB к ПК **для набора не подключают** — она не HID-клавиатура, только peripheral.

### Bluetooth

1. Питание левой (батарея или USB).
2. На ПК: найти **Corne** → сопряжение.
3. До 5 профилей BLE. Переключение — слой **ADJ** (зажмите нижний `mo 3` / крайний большой палец слева от середины правой половины — см. keymap):

| Клавиша (слой ADJ) | Действие |
|--------------------|----------|
| `&bt BT_SEL 0…3` | Профиль 0–3 |
| `&bt BT_CLR` | Очистить текущий профиль |
| `&out OUT_USB` / `OUT_BLE` / `OUT_TOG` | Принудительно USB / BLE / переключить |

Если ПК «не видит» клавиатуру после перепрошивки — `settings_reset` на левой, затем снова left UF2 и новое сопряжение.

## 3. Связь левой и правой половины

Половины связываются **между собой по BLE** (не через ПК).

### Как понять, что соединены

На OLED левой половины:

| Индикатор | Значение |
|-----------|----------|
| **`R:OK`** | Правая подключена к левой |
| **`R:--`** | Правая не на связи |

Дополнительно: нажатия на правой → символы на ПК; правая не видна как отдельное BT-устройство на ПК.

### Правая плата сейчас вне PCB

1. Питание правого nice!nano (USB или батарея).
2. Прошивка `corne_v3_right` из той же сборки Actions.
3. Через ~2–3 с на OLED слева → **`R:OK`**.

Если всегда `R:--`: `settings_reset` обеим → снова left+right; половины рядом; не прошить left на правый MCU.

## 4. Что показывает OLED

Кастомный экран = встроенный ZMK + флаг правой половины:

| Элемент | Смысл |
|---------|--------|
| `USB` / `BT1` | Выход на ПК (`?` = нет связи, `*` = не спарен) |
| `NN%` / `CHG NN%` | Заряд левой (CHG = на USB) |
| **`R:OK` / `R:--`** | Связь с правой половиной |
| Слой | `DEF` / `LOW` / `RAI` / `ADJ` |

Почему раньше был «белый шум»: режим `CUSTOM` не подтягивает шрифты/пул LVGL (в отличие от `BUILT_IN`). Это исправлено в `corne_v3_left.conf`.

Слои:

| № | Имя | Как попасть |
|---|-----|-------------|
| 0 | DEF | Базовый |
| 1 | LOW | Hold Space |
| 2 | RAI | Hold Enter (скролл трекбола) |
| 3 | ADJ | Hold `&mo 3` |

## 5. Слои и полезные клавиши

Кратко по смыслу (точные позиции — в `config/corne_v3.keymap`):

- **DEF** — буквы, home-row mods (A/S/D/F и J/K/L/;).
- **LOW** — цифры, RGB toggle, bootloader, ext_power, кнопки мыши.
- **RAI** — стрелки/nav, numpad-ish, скролл трекболом.
- **ADJ** — Bluetooth-профили и USB/BLE output.

Home-row mods: короткое нажатие = буква, удержание ≈ 200 ms = модификатор. Если «медленно печатаются» именно A/S/D/F — это hold-tap, не USB. Параметр `tapping-term-ms` у `hm` в keymap.

Обычный антидребезг сейчас **5 ms** (раньше было 20 ms).

## 6. Чем настраивать слои на ПК

### ZMK Studio (включено на левой половине)

1. Прошейте свежий `corne_v3_left` (сборка со snippet `studio-rpc-usb-uart`).
2. Подключите **левую** по USB.
3. Слой **ADJ** (`&mo 3`) → первая клавиша верхнего ряда: **`&studio_unlock`**.
4. Откройте [zmk.studio](https://zmk.studio/) (Chrome/Edge) или native app → Connect по USB.
5. Если клавиатура одновременно на USB и BLE — на ADJ выберите `&out OUT_USB`, иначе Studio может не отвечать.

После правок в Studio дальнейшие изменения `.keymap` в git **не** применятся, пока в Studio не сделаете Restore Stock Settings.

### Правка через git (без Studio)

1. Править `config/corne_v3.keymap` (и дубликат в щите).
2. Push → Actions → прошить UF2.

Визуальный редактор файла: [Keymap Editor](https://nickcoutsos.github.io/keymap-editor/).

### Что НЕ нужно

- QMK Toolbox / VIA / Vial — другой стек, к ZMK не подходит.

## 7. RGB

- Вкл/выкл: слой LOW → `&rgb_ug RGB_TOG`.
- Цепочка: 6 underglow, затем 21 per-key на половине.
- Если светят только первые LED — смотрите пайку/цепь, не софт счётчика (уже 27).

---

## 8. Трекбол (правая в PCB)

- Движение курсора на слое DEF (и других без scroll-mapper).
- На слое **RAI** (hold Enter) — скролл.
- Кнопка трекбола — ЛКМ (через split input).
- Настройка CPI/scale — см. [01-nastrojka.md](01-nastrojka.md).

---

## 9. Чеклист «всё живое»

1. Левая по USB печатает.
2. OLED: не шум; видны батарея, слой, **`R:OK`/`R:--`**.
3. Правая под питанием → через пару секунд **`R:OK`**; клавиша справа печатает.
4. BLE на батарее → ПК видит Corne.
5. (Опционально) трекбол / скролл на RAI.
6. Studio: unlock на ADJ → Connect на zmk.studio.

### Правая: `R:OK`, RGB есть, клавиши/трекбол мёртвые

Матрица и пины в прошивке совпадают с `pinout nicenano.txt`. Дальше — изоляция:

1. Прошить `settings_reset` на **обе** половины, затем снова left + right.
2. Прошить **`corne_v3_right_bare`** (без трекбола/I2C):
   - **Клавиши справа заработали** → виноват PAT912x/I2C (пайка D2/D3, addr 0x75, motion D8, питание VCC). Вернитесь на полный `corne_v3_right` после проверки железа.
   - **Клавиши всё ещё мёртвые** → сокет nice!nano / диоды / дорожки колонок (14,15,18,19,20,21), не флаги split (связь уже есть — `R:OK`).
3. Трекбол без курсора при живых клавишах: I2C и датчик; скролл только на слое **RAI**.

### Если снова «белый шум»

1. `settings_reset` → новый `corne_v3_left` из **этого** зелёного Build.
2. Если не ожило — `corne_v3_left_bare` и напишите (отключим CUSTOM снова).
